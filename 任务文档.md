# AI智能刷题学习助手 - 项目任务文档

## 项目概述
基于项目规划文档，采用**增量开发策略**：先构建可展示的框架骨架，再逐步往框架中添加功能模块，确保每个阶段都有可运行演示的成果。

---

## 开发策略

```
阶段1：搭建可展示框架（最小可运行系统）
    ↓
阶段2：往框架添加核心业务功能
    ↓
阶段3：集成AI能力
    ↓
阶段4：完善优化与工程化
    ↓
阶段5：交付与答辩准备
```

**核心原则**：每个阶段结束时，系统都必须是可运行、可演示的状态。

## 阶段1：构建可展示框架（最小可运行系统）

**目标**：搭建前后端完整骨架，实现基础页面路由、布局框架、静态数据展示，让系统可以启动运行并展示基本界面。

### T1.1.1 前端项目骨架搭建
- Vue 3 + Vite项目初始化
- Element Plus UI框架集成
- 配置路由系统（Vue Router）
- 配置状态管理（Pinia）
- 搭建整体布局框架（侧边栏导航 + 主内容区）
- 配置Axios请求封装

### T1.1.2 后端项目骨架搭建
- Python FastAPI项目初始化
- 配置项目目录结构（routers/services/models/schemas）
- 配置统一响应格式（code、message、data）
- 配置全局异常处理器
- 配置跨域CORS
- 配置Python logging日志体系

### T1.1.3 数据库设计与初始化
- 设计题目表（questions）
- 设计分类表（categories）
- 设计知识点表（knowledge_points）
- 设计错题表（wrong_questions）
- 设计刷题记录表（practice_records）
- 创建SQLite数据库初始化脚本
- 插入示例测试数据

### T1.1.4 前端基础页面框架
- 首页仪表盘页面（静态布局）
- 题库管理页面框架（表格布局）
- 刷题练习页面框架（题目展示布局）
- 错题本页面框架（列表布局）
- 学习统计页面框架（图表占位）

### T1.1.5 前后端联调验证
- 后端健康检查接口
- 前端调用后端接口测试
- 解决跨域问题
- 验证系统可正常启动运行

**阶段1里程碑**：系统可启动运行，前端页面可正常访问跳转，前后端可正常通信

## 阶段2：往框架添加核心业务功能

**目标**：在可运行的框架基础上，逐步添加题库管理、刷题练习、错题本、学习统计等核心业务功能。

- 题目新增接口
- 题目编辑接口
- 题目删除接口
- 题目分类管理接口
- 前端题库管理页面对接
- 手动录入题目表单
- 批量导入题目功能

### T1.2.2 刷题练习功能
- 按知识点选题接口
- 随机选题接口
- 顺序选题接口
- 答题提交与判分接口
- 错题自动收录逻辑
- 前端刷题页面对接
- 答题交互体验优化

### T1.2.3 错题本功能
- 错题列表查询接口
- 错题详情接口
- 错题重做接口
- 错题标记/取消标记接口
- 错题删除接口
- 前端错题本页面对接

### T1.2.4 学习统计功能
- 刷题量统计接口
- 正确率统计接口
- 知识点掌握度统计接口
- 学习时长统计接口
- 前端统计页面对接
- ECharts图表集成（柱状图、折线图）

**阶段2里程碑**：基础刷题全链路跑通，所有核心业务功能可正常使用

## 阶段3：集成AI能力

**目标**：封装AI服务层，逐步集成4项核心AI能力，实现智能出题、错题解析、薄弱点总结、模拟组卷。

### T1.3.1 AI服务层封装
- 统一AI调用接口设计
- 豆包大模型API对接
- 通义千问API对接（备选）
- 超时重试机制
- 异常降级兜底（AI不可用时返回本地题库）
- 输出格式校验与解析

### T1.3.2 AI智能出题功能
- AI出题Prompt设计与优化
- 按知识点、题型、难度生成题目接口
- AI生成题目自动入库
- 前端AI出题交互页面

### T1.3.3 错题AI解析功能
- 错题解析Prompt设计与优化
- 答题错误后触发AI解析接口
- 生成拓展解析、知识点拆解
- 易错点提示展示
- 前端AI解析展示页面

### T1.3.4 薄弱点总结功能
- 薄弱点分析Prompt设计与优化
- 拉取历史刷题数据分析接口
- 生成知识点掌握度排名
- AI个性化学习建议
- 前端薄弱点分析页面

### T1.3.5 AI模拟组卷功能
- 模拟组卷Prompt设计与优化
- 指定考试范围、题型占比、难度分布
- 自动生成完整试卷接口
- 支持答题与评分
- 前端模拟考试页面

**阶段3里程碑**：所有AI功能上线，全业务流程闭环可运行
